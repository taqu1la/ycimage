Platform Statement

当前支付、数据与内容处理方式的基础合规说明

YCImage 是一个面向图片与视频模板生成的在线平台。当前方案中，前端与业务系统可部署在 Cloudflare，支付通过 Stripe Checkout 完成，支付结果通过 Stripe Webhook 通知平台，再由平台为用户发放积分、会员权益或订单状态更新。

支付页面由 Stripe 托管 业务数据可托管于 Cloudflare D1 / R2 积分与权益由平台后台账本控制

Payment

银行卡与支付安全

当你使用 Stripe Checkout 付款时，银行卡号、有效期、CVC 等敏感支付信息在 Stripe 托管的支付页面录入和处理。YCImage 不应直接存储完整银行卡信息，也不应在自己的前端或数据库中保存完整卡号。

Storage

图片、视频与账号数据

用户账号、积分、订单、任务记录可存放在数据库服务中；用户上传图片、生成结果和视频文件可存放于对象存储。你上传的内容将用于完成生成、下载、售后、风控和争议处理。

Risk Control

争议、拒付与审计

为处理拒付、退款、账户滥用和违规内容，平台可能保留订单号、付款状态、IP、账号标识、积分流水、生成记录、下载记录及协议同意记录。

Policies

上线前至少需要的政策页

建议在正式收款前公开展示用户协议、隐私政策、退款政策和内容规则，并在注册、下单、充值和上传入口可见位置提供访问链接。

当前方案说明

你现在这套部署结构下，建议对外这样声明

模块 建议声明口径
网站与 API 网站页面、业务接口、订单逻辑、积分账本、任务状态等由 YCImage 平台负责维护，可运行在 Cloudflare Pages、Workers 或其他云服务上。
数据库与对象存储 账号、积分、订单、任务记录可存放在数据库中；上传图片、生成图片、视频文件和静态资源可存放在对象存储中，用于生成、下载、售后与风控。
支付处理 支付通过 Stripe Checkout 等第三方支付服务完成。用户支付时会跳转到 Stripe 托管的结账页面，支付机构会处理银行卡及其他支付方式信息。
支付回调 支付成功后，Stripe 会通过 Webhook 通知平台。平台以服务端校验后的订单结果为准，为用户加积分、开通会员或更新订单状态。
银行卡信息 平台不主动收集、不展示、不保存完整银行卡号、CVC 或有效期。若未来引入卡片保存功能，也应仅使用支付服务商返回的 token、payment method ID 或掩码卡信息。
Important Reminder

正式上线前你还需要补全的关键信息

  • 运营主体名称：个人、个体户或公司名称，建议写入所有政策页页脚。
  • 有效联系方式：客服邮箱、售后邮箱、法务或投诉邮箱，至少保留一个可访问邮箱。
  • 退款处理时限：例如“收到申请后 5-10 个工作日内处理”。
  • 数据保留周期：例如账号删除后保留订单和风控数据的最短必要期限。
  • 未成年人规则：是否允许未成年人使用，是否需要监护人同意。
  • 适用法域：你打算以哪个国家或地区主体运营、签约和收款。
